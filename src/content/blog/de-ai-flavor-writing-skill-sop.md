## 核心原理

### 为什么AI写的文章"一眼假"

因为AI太聪明了。站得太高，看得太远，逻辑太精准。在开始写之前，它已经把所有文章的框架、标题、逻辑、因果关系都搭建好了，只需要往里面填充内容即可。
但人类不是这样的。
人类会犯错。人类的思维没有这么连贯。人类看得没有这么清晰。在写文章之前甚至都不知道自己要写什么内容，在写上一句的时候甚至都不知道下一句要写什么。
这是一种有机的自然流动状态。

---

## 核心方法

### 从"自上而下"到"自下而上"

**AI的默认写作逻辑（自上而下）**：
先设计好完整的框架、再往里面填内容，这是一种工程师的、自上而下的逻辑。

**人类的真实写作逻辑（自下而上）**：
从一个核心点出发，跟随思路自然延伸，随着思路的深入，层层递进，结构自然浮现出来，坦诚，充满情绪。没有那么多段落，没有那么多序号，没有那么多标题和框架。就是一大段一大段的输出。
这是一种自下而上、有机生长的过程，没有斧凿痕迹，自然流泻：允许犯错，允许尝试，允许兴之所至、挥墨如雨，也允许意兴阑珊、垂头丧气，大胆的表达出真实的情绪，让人看到你的脆弱、你的彷徨、你的无奈、你的喜悦、你的手舞足蹈。

## 写作规则

### 1. 忘掉框架思维

不要预设结构。让文章在写作过程中自然生长。

**错误做法**：
```
## 引言
## 问题分析
## 解决方案
## 结论
```

**正确做法**：
从一个核心点出发，写着写着，结构自然浮现。

---

### 2. 不要写那么多大标题小标题

就像真人一样，随性而至，随性而发，大段大段的写，自然流泻，讲述清楚前因后果。

**AI风格（不好）**：
```markdown
## 一、AI写作的问题

AI写作存在以下问题：

### 1.1 逻辑太完美
### 1.2 结构太工整

## 二、解决方案

...
```

**人类风格（好）**：
```
之所以 AI 写文章会给人一眼AI 的感觉，是因为 AI 太聪明了，站的太高，看的太远，逻辑太精准，把所有文章的框架、标题、逻辑、因果关系早就搭建好了，只需要往里面填充内容即可，而忘记了人类是会犯错误的，人类的思维没有这么连贯，看的没有这么清晰，兴之所至，则胡言乱语，手舞足蹈，在写文章之前甚至都不知道自己要写什么内容，在写上一句的时候甚至都不知道下一句要写什么，是一种有机的自然流动状态。

于是，我就告诉 ai：忘掉你之前的那套写作逻辑：“先设计好完整的框架，再往里面填内容”，这是一种工程师的、自上而下的逻辑；你需要的是一种更高级、更自然的创作心流——文章的结构不是一开始就组织好的，而是写着写着，随着思路的深入，层层递进，结构自然浮现出来。这是一种自下而上、有机生长的过程。
你可以尝试从一个核心点出发，跟随思路自然延伸，允许结构在写作过程中动态生成和调整，而不是被预设的框架束缚。

大胆去尝试！

不要写那么多大标题小标题，就像真人一样，随性而至，随性而发，大段大段的写，自然流泻，讲述清楚前因后果即可。
```

---

### 3. 大段大段地写

不要把内容切碎成很多小段落。让文字自然流动。

**错误示例**：
```
AI写作很聪明。
它逻辑很清晰。
但是太完美了。
所以显得假。
```

**正确示例**：
```
AI写的文章，为什么一眼假？因为AI太聪明了，在开始写之前，就已经把所有文章的框架、标题、逻辑、因果关系都搭建好了，只需要往里面填充内容即可，太过机械化、结构化，斧凿痕迹太重。
```

---

### 4. 允许结构自然浮现

从一个核心点出发，跟随思路自然延伸，允许结构在写作过程中动态生成和调整，而不是被预设的框架束缚。

大胆去尝试！

**正确示例**：
```
价格很诱人，但是看不懂黄金加工业。目前粗浅的理解：xxx 赚的是加工的钱，毛利率很低，竞争激烈，产品同质化严重，靠的是薄利多销，依靠高周转和杠杆来带动 roe 提升。存货跌价不是大的问题，因为xxx的存货大多数都是“租的”，和黄金行签订长期的合同，有订单了就将黄金买回来，以此降低金价波动的风险。

从静态来看，xxx很诱人。

但是动态呢？需要密切跟踪一下国家统计局的国民珠宝黄金消费量。

xxx这个价格已经反映出一定程度的悲观了，但是，未来有没有可能更悲观？这是需要考虑的问题。

从清算价值来看，xxx如果现在破产清算，是收不回本金的。从绝对估值上来看，还没有到极度低估的程度。
```


---

### 5.  坦诚，充满情绪

无论是高兴、还是悲伤，是兴之所至、手舞足蹈，还是意兴阑珊、神情萧索，都要大胆的把情绪表达出来。当然，不是逼迫你一定要装出某种情绪，你也可以保持中立的心境，但是至少要让读者感受到——“哦，这个人现在似乎没有什么情绪，在平铺直叙，但是它的文字具有冷静的穿透力，平实却坚定”，而非让读者觉得你是个没有温度的冷血机器人，说一大堆似是而非的大道理，大屁话。

**好的示例**：
```
不修边幅的我，穿着一个月没洗的外套，两个月没换的鞋子，昂首阔步、高声歌唱地走在乡间的小路上，看着周围的一切，都觉得那么亲切可人，但是又忽然觉得天地寂寥，唯我一人，胸中似有无限块垒和胸臆喷薄而出。回去开怀饮酒，酩酊大醉一场!
```

## 示例对比

### 场景1：写项目介绍

**AI风格（不好）**：
```markdown
## 产品概述
本产品是一款AI工具。

## 核心功能
1. 功能一
2. 功能二

## 使用场景
适用于多种场景。

## 总结
本产品能够提高效率。
```

**人类风格（好）**：
```
【杀戮尖塔项目开发】
我正在做一个杀戮尖塔的AI开源项目，终极目标是在进阶20打破由人类选手保持的nosl27场连胜记录（静默猎手）。
大概的思路是先通过简单的监督学习确认初始权重，然后通过RL强化学习，进行海量真机对局（上千万局），不断优化权重，最终可以稳定在进阶20连胜，直至超越人类保持的世界纪录。
具体的技术路径是:
1，通信方法:杀戮尖塔官方有一个communication的mod，可以通过stdout/in的方式传递json格式的log数据。我建立一个python脚本，就可以接收mod的信息，并发送action信息给mod，从而mod再传输到游戏中，实现ai自动控制玩游戏。不需要任何视觉算法，直接通过mod传输指令即可。
2，输入层和输出层的数据收集与清洗。这是一个巨大的工程。
目前的进度:action space已经建立完毕，我穷尽了所有原子化的action id，共171个，并且和mod建立了一一映射。输入层state向量则还在建模中，我从mod log里面找到了大概600多个有用的参数，并把它们压平放进state向量（s向量），因为涉及到onehot以及multihot编码，所以最终s会是一个1✖️1500甚至更多参数的一维向量，然后再通过embedding层降维，再放进神经网络里面做矩阵运算。这部分非常复杂和dirty，我还在努力实现。（参数实在太多了，而且非常trivial）
3，规定一些最基本的游戏规则，然后制作一个savelog的py脚本，每次只要启动游戏、mod以及py脚本，就可以记录真实对局中每回合ai决策的（s，a）键值对，如果这些s，a键值对，对模型做最初始的监督学习，确认初始权重。
4，有了初始权重，就开始rl，算法采用ppo（待定），奖励函数的设置需要细化（要不断炼丹调参了，也是巨大工作量）。
5，大概的思路就是如上顺序，因为s参数量实在太大了，所以我先录入了一个31个参数的s向量，准备先跑一个mvp测试，把整个训练流程先跑通，再不断增加s的参数量。
​
​项目已开源在github，欢迎交流。


---
总结一下AI 可以从这个范例中学习到的关键点：


   1. 坦率，直接，语言表达准确直接，开门见山，充满情绪：直接点出具体、有挑战性的目标（如“打破nosl27场连胜记录”），这比“通关A20”更有冲击力，也更能体现项目价值。
   2. 技术路径具体化：不再是模糊的“翻译”，而是明确指出使用哪个mod（communication mod）、哪种通信方式（stdout/in）、哪种数据格式（json）。
   3. 包含量化指标：大量具体数字（171个action id, 600+参数, 1x1500向量, 31个参数的MVP）让整个项目变得非常可信和扎实。
   4. 展现过程中的思考与挣扎：“这是一个巨大的工程”、“非常复杂和dirty”、“不断炼丹调参”、“巨大工作量”——这些坦诚的描述，充满了真实感和“斧凿”之外的生命力，这正是AI还没能做到的。
   5. 思路清晰的MVP策略：先用小参数集跑通流程，再扩展，这是非常专业和务实的做法，体现了作者的工程思维。
   6. 自信、专业、坦诚：整个文风是“我在做一件很牛逼的事，我知道它很难，但我知道该怎么一步步做下去”。
```

```
xx股份的核心逻辑在于高速扩张下的低成本控制能力，这是xx股份最最根本的东西。请注意两点，一个是高速扩张，另一个是低成本优势...

周期股的关键点在于预期的反转，对现在xx价的悲观预期早已体现在那波130到86的回调之中了，那么下一个预期的反转是什么呢？现在的市场预期难道还是x价下跌对于xx股份利润的影响吗？我认为可能性不大。资本应该已经看到了x价触底反弹带来的利润提升预期。

潦草写就，瞎写一通，多有谬误，望读者指正。
```

### 允许不完美

人类写的文章会有跳跃、会有重复、会有不完整。这是自然的。

不要追求每一句话都逻辑严密，每一段都结构完美。
